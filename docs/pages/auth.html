How it works

1. Authorization Request

The app redirects the user to the Microsoft identity platform endpoint. The user signs in and consents to permissions.

2. Token Exchange

The platform returns an authorization code. The app exchanges this code for an access token using its client secret (backend).

3. API Access

The app uses the access token to call Microsoft Graph API on behalf of the user.
